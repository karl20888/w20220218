Vue 3 #

历程 #

  • 2013年12月开始
  • 2015年9月 1.0 预览版发布
  • 2016年6月 2.0 预览版发布
  • 2021年6月 2.6.14 最新版本
  • 2020年1月 3.0 预览版
  • 2020年9月18日 3.0 正式版
  • 2021年6月8日 3.1 正式版
  • 2021年8月10日 3.2 正式版
  • 2021年12月12日 3.2.26 最新版

值得注意的新特性 #

Vue 3 中一些需要关注的新功能包括：

  • 组合式 API
  • Teleport
  • 片段
  • 触发组件选项
  • 来自 @vue/runtime-core 的 createRenderer API，用于创建自定义渲染器
  • 单文件组件组合式 API 语法糖（<script setup>）
  • 单文件组件状态驱动的 CSS 变量（<style> 中的 v-bind）
  • SFC <style scoped> 现在可以包含全局规则或只针对插槽内容的规则
  • Suspense

注意： #

Vue3 将不再兼容任何 IE 版本！

优势 #

性能提升 #

响应式系统提升 #

Vue 3 使用 Proxy 对象重写响应式。不需要初始化的时候遍历所有属性，另外有多层属性嵌套的话，只有访问某个属性的时候，才会递归处理下一级属性。

  • 可以监听动态新增的属性
  • 可以监听删除的属性
  • 可以监听数组的索引和 length 属性

编译优化 #

  • 优化编译和重写虚拟 DOM，让首次渲染和更新 DOM 性能有更大的提升，Vue 3 标记和提升所有静态根节点，diff 的时候只比较动态节点内容。

  • 片段，模板里面不用创建唯一根节点，可以直接放同级标签和文本内容。

  • 静态标记，跳过静态节点，直接对比动态节点，缓存事件处理函数。

Composition API 与 Options API #

Options API #

  • 包含一个描述组件选项（data、methods、props 等）的对象 options。
  • API 开发复杂组件，同一个功能逻辑的代码被拆分到不同选项。
  • 使用 mixin 重用公用代码，需维护命名冲突，数据来源不清晰。

Composition API #

  • 基于函数的 API，可以更灵活地组织组件的逻辑。
  • 解决 Options API 在大型项目中不好拆分和重用的问题。

Proxy 与 Object.defineProperty #

  • Proxy 性能更好，并且未来作为新功能持续交给浏览器去不断优化。
  • Proxy 可以拦截属性的访问、赋值、删除等操作。
  • 可以劫持的操作有 13 种。

编译 #

Vue 3 标记和提升所有的静态节点，diff 算法只需要对比动态节点内容。

Element Plus #

一套为开发者、设计师和产品经理准备的基于 Vue 3 的桌面端组件库。

随着 Vue 3 的发布，Element UI 也在同年开始进行升级重构，新项目定名为 Element Plus，并在 2020 年年底发布了第一个预览版，经历了一整年近百个版本后，目前已经发布到 1.2.0-beta.6 版，但始终没有真正的 release 版。不过相信 Element 团队开发出的产品的稳定性禁得住考验。
