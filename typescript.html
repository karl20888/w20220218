TypeScript（JavaScript 的超集） #

简介 #

TypeScript 是 JavaScript 类型的超集，它可以编译成纯 JavaScript。

TypeScript 扩展了 JavaScript，为它添加了类型支持。（2021年1月18日）

TypeScript 是微软开发的一个开源的编程语言，通过在 JavaScript 的基础上添加静态类型定义构建而成。TypeScript 通过 TypeScript 编译器或 Babel 转译为 JavaScript 代码，可运行在任何浏览器，任何操作系统。

背景 #

TypeScript 起源于使用 JavaScript 开发的大型项目 。由于 JavaScript 语言本身的局限性，难以胜任和维护大型项目开发。因此微软开发了 TypeScript ，使得其能够胜任开发大型项目。

特点 #

随着 ES6 标准的到来，其实当年的 TS 的绝大多数新特性已经“没啥卵用”了！

  • 类型！类型！类型！
  • 接口、枚举、泛型其实也是类型。
  • 命名空间、装饰器、Mixins

优势 #

  • 强类型规范
  • 减少运行时错误
  • 适合大项目
  • 编辑器代码提示优化
  • 利于代码重构、扩展
  • 注释文档

劣势 #

  • 类型的引入会让我们书写起来极不适应！
  • 我们会花一些时间去描述类型！
  • 代码量增加，并且很难控制代码质量（对我们来说）！
  • Vue2.0 并没有和 TypeScript 很好的结合！

趋势 #

  • 任何使用 JavaScript 编写的应用，都应该使用 TypeScript 来实现。
  • Vue3.0 是用 TypeScript 重构的。
  • element-plus 是用 TypeScript 重构的。
  • VSCode 是用 TypeScript 开发的。

总结：其实这些趋势跟我们也没太大关系，如果我们大部分代码都是在写页面，那 TypeScript 的确是没必要。
